## Overview

AI NewsVault is an AI-powered news intelligence platform that transforms the overwhelming volume of
daily news into a curated, contextualised, and analytically enriched reading experience. Rather than
presenting raw headlines, it aggregates news from configurable sources via RSS and news APIs, clusters
related stories across outlets to identify the same event from multiple perspectives, summarises each
cluster into a concise brief with source attribution, and performs editorial analysis: bias detection,
sentiment classification, entity extraction, and fact-checkable claim identification.

The platform is designed around the principle that news literacy requires context that individual articles
rarely provide. When a story breaks, what do different sources emphasise? What is the historical context?
Who are the key actors and what are their stated interests? AI NewsVault answers these questions
automatically — enriching each story cluster with relevant historical background, key entity profiles,
and sentiment comparison across sources.

A personalisation layer allows users to define interest profiles (technology, geopolitics, science,
economics) with configurable topic weights, ensuring the most relevant clusters surface first without
creating a pure filter bubble — a small percentage of the feed is always deliberately diverse.

---

## Motivation

Modern news consumption has two failure modes: information overload (too many sources, too much content)
and filter bubbles (algorithmic curation that reinforces existing beliefs). AI NewsVault was designed to
address both simultaneously — reducing volume through intelligent clustering and summarisation while
increasing diversity through deliberate cross-source perspective presentation.

## Architecture

```
News Sources (RSS, NewsAPI, GDELT)
        │
  Article fetch and deduplication
        │
  Semantic clustering (sentence-transformers + HDBSCAN)
        │
  ┌──────────────────────────────────────────────┐
  │  Per-cluster analysis:                       │
  │  ├── LLM summarisation (3-sentence brief)   │
  │  ├── Sentiment analysis (VADER / FinBERT)   │
  │  ├── Entity extraction (spaCy NER)           │
  │  └── Source bias classification             │
  └──────────────────────────────────────────────┘
        │
  Personalised ranking (interest profile weights)
        │
  Streamlit news dashboard
```

## Features

### Multi-Source News Aggregation
Fetch articles from 50+ configurable RSS feeds, NewsAPI endpoints, and the GDELT project database — covering global news in 15+ languages with automatic translation.

### Semantic Story Clustering
Group related articles from different sources into story clusters using sentence-transformer embeddings and HDBSCAN clustering, eliminating duplicate coverage of the same event.

### AI Story Briefs
Three-sentence neutral summaries of each story cluster generated by an LLM, with in-line source citations for every claim — giving maximum information density with minimum reading time.

### Cross-Source Sentiment Analysis
Sentiment scores (positive/neutral/negative) for each source's coverage of the same story, visualised as a comparison bar chart to reveal framing differences.

### Named Entity Extraction
Automatic identification of people, organisations, locations, and events mentioned in each cluster, with clickable entity profiles linking to recent coverage.

### Media Bias Classification
Each news source labelled with its known media bias rating (using AllSides or Ad Fontes Media data) and displayed alongside articles for transparent source context.

### Personalised Interest Feed
User-configurable topic weights (technology, climate, politics, science, markets) that re-rank the cluster feed while maintaining a diversity floor of non-preferred topics.

### Fact-Checkable Claim Extraction
Identify specific factual claims in articles (statistics, attributed quotes, event assertions) and flag them for potential fact-checking against known databases.

## Tech Stack

| Library / Tool | Role | Why This Choice |
|---|---|---|
| **sentence-transformers** | Semantic clustering | Article embedding for similarity-based story grouping |
| **HDBSCAN** | Clustering algorithm | Density-based clustering without pre-specifying cluster count |
| **OpenAI / Gemini** | Story summarisation | Neutral, citation-rich cluster brief generation |
| **spaCy** | NLP/NER | Named entity extraction and relationship identification |
| **VADER / FinBERT** | Sentiment analysis | General and financial news sentiment classification |
| **feedparser** | RSS ingestion | RSS/Atom feed parsing and normalisation |
| **Streamlit** | News dashboard | Story feed, sentiment charts, entity panels |
| **pandas** | Data pipeline | Article deduplication, cluster management, history |

## Getting Started

### Prerequisites

- Python 3.9+ (or Node.js 18+ for TypeScript/JavaScript projects)
- A virtual environment manager (`venv`, `conda`, or equivalent)
- API keys as listed in the Configuration section

### Installation

```bash
git clone https://github.com/Devanik21/AI-NewsVault.git
cd AI-NewsVault
python -m venv venv && source venv/bin/activate
pip install streamlit sentence-transformers hdbscan spacy feedparser openai \
            vadersentiment pandas plotly requests python-dotenv
python -m spacy download en_core_web_sm
echo 'OPENAI_API_KEY=sk-...' > .env
echo 'NEWS_API_KEY=your_key' >> .env
streamlit run app.py
```

## Usage

```bash
# Launch news dashboard
streamlit run app.py

# Fetch and cluster today's news
python fetch_news.py --sources config/sources.yaml --cluster True

# Generate story brief for a cluster
python summarise_cluster.py --cluster_id 42

# Export digest as email-ready HTML
python export_digest.py --format html --output daily_digest.html
```

## Configuration

| Variable | Default | Description |
|---|---|---|
| `NEWS_API_KEY` | `(required)` | NewsAPI.org key for structured news data |
| `OPENAI_API_KEY` | `(required)` | LLM key for story summarisation |
| `SOURCES_CONFIG` | `config/sources.yaml` | RSS feed sources configuration file |
| `REFRESH_INTERVAL_MIN` | `30` | News refresh interval in minutes |
| `CLUSTER_MIN_SIZE` | `2` | Minimum articles per cluster (HDBSCAN parameter) |
| `DIVERSITY_FLOOR` | `0.15` | Fraction of feed reserved for non-preferred topics |

> Copy `.env.example` to `.env` and populate required values before running.

## Notes

Media bias ratings are sourced from AllSides and Ad Fontes Media, which themselves have methodological debates. Bias labels should be interpreted as a starting point for critical reading, not as definitive verdicts. Sentiment analysis on political topics may be less reliable than on financial or scientific news.
